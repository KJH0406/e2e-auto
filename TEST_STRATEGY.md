# E2E 테스트 전략

Claude Code 토큰 소비 최소화를 목표로 하는 E2E 테스트 실행 전략입니다.

---

## 원칙

- Snapshot은 꼭 필요할 때만 사용한다
- 실패 재시도는 최대 1회로 제한한다
- 요소를 찾지 못하면 즉시 중단하고 부분 코드를 제공한다

---

## 실행 흐름

### 1단계: 시나리오 확인

사용자가 테스트 시나리오를 제공하지 않으면 먼저 요청한다.

시나리오에는 다음이 포함되어야 한다:
- 테스트 대상 URL
- 순서대로 나열된 실행 단계
- 각 단계에서 기대하는 결과

### 2단계: 인증 상태 확인

| 상태 | 처리 |
|------|------|
| `auth.json` 유효 | 바로 3단계로 진행 |
| `auth.json` 만료 또는 없음 | 로그인 실행 → `auth.json` 저장 → 3단계로 진행 |

### 3단계: 초기 Snapshot 1회 실행

페이지 진입 후 snapshot을 **1회** 찍어 시나리오에 필요한 **모든 요소를 한 번에 식별**한다.

- 요소가 모두 식별되면 → 4단계로 진행
- 일부 요소가 없으면 → 식별된 요소까지만 진행하고 이후는 순차 처리

### 4단계: 시나리오 순서대로 실행

식별된 요소를 기반으로 시나리오를 순서대로 실행한다.

**Snapshot 추가 촬영 기준:**
- 페이지가 전환된 경우
- 현재 snapshot에서 다음 단계 요소를 찾지 못한 경우

같은 페이지 내에서는 기존 snapshot을 재활용한다.

### 5단계: 요소 탐색 실패 처리

요소를 찾지 못하면 다음 순서로 처리한다:

```
요소 탐색 실패
→ 현재 페이지 snapshot 1회 재시도
→ 요소 발견 시 계속 진행
→ 그래도 실패 시 즉시 중단
  → 현재까지 완성된 코드 제공
  → 실패한 단계와 이유 안내
  → 이후 단계는 요소 확인 필요 명시
```

---

## Snapshot 사용 규칙 요약

| 상황 | Snapshot |
|------|----------|
| 시나리오 시작 | 1회 (전체 요소 식별) |
| 페이지 전환 | 1회 |
| 요소 탐색 실패 | 최대 1회 재시도 |
| 동일 페이지 내 다음 요소 | 기존 snapshot 재활용 |

---

## 실패 시 결과물 형식

```
## 완료된 단계 (1 ~ N)
[완성된 테스트 코드]

## 중단된 단계 (N+1)
- 실패 이유: [요소명 / 셀렉터] 를 찾지 못함
- 이후 단계: 요소 확인 후 진행 필요
```

---

## auth.json 갱신 기준

- 로그인 후 `auth.json`을 저장한다
- 테스트 실행 시 인증 오류가 발생하면 갱신이 필요한 시점으로 판단한다
- 갱신은 2단계(인증 상태 확인)에서 자동으로 처리된다
